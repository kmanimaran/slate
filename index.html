    Introduction

    Welcome to the WeLink API. You can use our API to access API endpoints that provide social media data based on locations, people and keywords or topics of interest.

    You can view API examples in the dark area to the right.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "https://welink.com/api/v3/endpoint?apiKey=XXXX"

    Make sure to replace XXXX with your API key.

    WeLink uses apiKey parameter to allow access to the API. You can get an API key by contacting us - contact@welink.com.

    WeLink expects the apiKey parameter to be included in all API requests to the server as a parameter:

    apiKey=<XXXX>

    You must replace XXXX with your personal API key.

    This endpoint retrieves the most recent social media posts from either locations or people (social media usernames or full name) or keywords

    HTTPS Request

    GET https://welink.com/api/v3/getLiveData.jsp

    Query Parameters

    Parameter Required Default Description
    source true Possible values are TWIITER, INSTAGRAM, FOURSQUARE, YOUTUBE, FLICKR, VK, WEIBO, EVENTBRITE
    locations optional Comma separated value of locations. Each location can be a circle or a rectangle or a polygon. Multiple locations are separated by ’,’. Each location is represented as follows: Circular locations are represented as ‘C:latitude:longitude:radius (in meters)’. For e.g. C:C:40.758899:-73.9873197:100. Rectangular locations are represented as 'R:<South west point latitude>:<South west point longitude>:<North east point latitude>:<North east point longitude’>. Polygons are represented as 'P:<Point 1 latitude>:<Point 1 longitude>:<Point x latitude:Point x longitude>’.
    username optional Returns data based on social handle or Full name. Supports twitter handles, instagram handles, Full names and email addresses. Format for twitter is @username. Format for instagram is in/username. Results are applied only to one of location or username or keyword parameters if multiple parameter values are present. locations parameter takes precedence over username in that case.
    keyword optional Returns data based on keywords from all over the globe. Results are applied only to one of location or username or keyword parameters if multiple parameter values are present. locations and username parameter takes precedence over keyword.
    limit optional 100 Number of posts to return.
    offset optional Offset from where to return the posts. offset is an absolute number and does not denote page number.
    filter_from optional Start date of filter. Format MM/DD/YYYY.
    filter_to optional End date of filter. Format MM/DD/YYYY.
    filter_keyword optional Keywords to filter the results on.
    Remember â€” make sure to pass a valid apiKey parameter!

    This endpoint retrieves all kittens.

    HTTPS Request

    GET https://welink.com/api/v3/getTrueId.jsp

    Query Parameters

    Parameter Required Default Description
    username true This parameter can either be social media username, email address or a full name. Social media handles need to be passed in the following format: Twitter - @username or tw/username or twitter.com/username. Instagram - in/username or instagram.com/username. Email - username@domain.com. Name can be entered without any formatting.
    Remember â€” Any value for username parameter that doesn’t follow the formatting for social media usernames or email will be treated as a full name.

    Errors

    WeLink API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks :)
    401 Unauthorized – Your API key is wrong or not present
    403 Forbidden – The endpoint requested is hidden for administrators only
    404 Not Found – The specified endpoint could not be found
    405 Method Not Allowed – You tried to access an endpoint with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The endpoint requested has been removed from our servers
    429 Too Many Requests – You’re sending too many requests! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
